# Social Post Image (Cross-Platform Quick Reference)

Quick reference for standalone social media images across platforms. For deeper per-platform guidance, see:
- `linkedin-post.md` — LinkedIn single-image posts
- `linkedin-carousel.md` — LinkedIn carousel/document posts (highest B2B engagement format)
- `linkedin-banner.md` — LinkedIn company and personal banners
- `twitter-x.md` — Twitter/X posts, banners, algorithm context
- `social-og.md` — Open Graph images for link sharing

## Dimensions

| Platform | Recommended size | Aspect ratio | Notes |
|---|---|---|---|
| **LinkedIn (single image)** | 1200 x 627 px | 1.91:1 | Same as OG; most common for link-style posts |
| **LinkedIn (square)** | 1080 x 1080 px | 1:1 | Carousel slides, standalone visual posts |
| **LinkedIn (portrait)** | 1080 x 1350 px | 4:5 | Takes up more feed space; good for engagement |
| **Twitter/X (single image)** | 1600 x 900 px | 16:9 | Optimal for timeline display |
| **Twitter/X (square)** | 1080 x 1080 px | 1:1 | Works well but less feed real estate |
| **Instagram (square)** | 1080 x 1080 px | 1:1 | If cross-posting |
| **Instagram (portrait)** | 1080 x 1350 px | 4:5 | Maximum feed presence |

**Default recommendation**: 1200 x 675 px (16:9) works across LinkedIn and Twitter without cropping.

**For carousel posts** (LinkedIn, Twitter): 1080 x 1080 px per slide.

## Design guidelines

### Text
- **Headline**: 60–80px at 1200w — readable on mobile without zooming
- **Supporting text**: 32–40px at 1200w
- **Max 15 words total** on the image — social images are glanced at, not read
- **If the text is the content** (quote cards, stats, announcements): use large, bold type as the primary visual element

### Composition
- **Mobile-first** — most social feeds are viewed on phones. Design for a ~400px wide viewport
- **One message per image** — the post caption provides context; the image should communicate one thing instantly
- **High contrast** — social feeds are visually noisy. The image needs to pop against a white or dark feed background
- **Faces and people draw attention** — if appropriate, photos of real team members outperform abstract graphics for engagement

### Post type templates

| Post type | Recommended approach |
|---|---|
| **Product announcement** | Screenshot/mockup of the feature + bold headline |
| **Blog promotion** | Use the blog cover thumbnail (already 16:9) — no need to create a separate graphic |
| **Stat / metric** | Large number as the hero element + brief context |
| **Quote** | Speaker photo + quote text in large type |
| **Event / webinar** | Date + title + speaker headshots |
| **Carousel (educational)** | Consistent template across slides; title slide + 4-6 content slides + CTA slide |

### Brand elements
- **Inkeep logo** — subtle placement (corner). The social profile already identifies the brand
- **Consistent color palette** — builds brand recognition across posts
- **Watermark the bottom-right** if the image is likely to be screenshotted and reshared without attribution

### What to avoid
- Text in the outer 5% (some platforms add rounded corners or overlays)
- Relying on the image alone to convey the message (always pair with post copy)
- Repurposing a wide blog cover as a square post without redesigning (crops awkwardly)
- Busy backgrounds behind text (legibility first)
- Using platform-specific UI mockups (Slack screenshots, terminal output) without ensuring they're readable at mobile sizes
