# LinkedIn Carousel / Document Post

Standard for carousel (document) posts on LinkedIn — the **highest-engagement B2B format**.

## Why carousels first

Carousels are the top-performing format on LinkedIn by a wide margin:

| Format | Engagement Rate | Source |
|---|---|---|
| **Carousel** | **6.60% — 21.77%** | SocialInsider (1M posts) / Buffer (52M posts) |
| Document (PDF) | 5.85% | SocialInsider |
| Video | 5.60% | SocialInsider |
| Single image | 4.85% | SocialInsider |

Carousels hold attention **15-20 seconds** per post vs 8-10 seconds for single images. They generate **2x higher CTR** than single images.

## Dimensions

| Format | Dimensions | Aspect Ratio | Best for |
|---|---|---|---|
| **Square (recommended)** | 1080 x 1080 px | 1:1 | Consistent cross-device display |
| **Portrait** | 1080 x 1350 px | 4:5 | Maximum feed real estate on mobile |

| Property | Value |
|---|---|
| **File format** | PDF (recommended). Also PPT, PPTX, DOC, DOCX |
| **Max file size** | 100 MB total |
| **Max slides** | 300 (technical), **5-10 optimal** |
| **Safe zone** | Keep key content within center ~880 x 880 px (for 1080 format) |
| **Edge padding** | 80-100 px on all sides |

## Typography tiers at 1080×1080

| Role | % of canvas height | Target range | Scale step |
|---|---|---|---|
| **Heading** | 5-7% | 54-76px | 64 |
| **Subtitle** | 2.5-3.5% | 28-36px | 28 or 36 |
| **Body** | 2-3% | 24-32px | 28 |
| **Badge** | 1.3-1.8% | 14-19px | — |

Square formats use lower height percentages because the canvas is taller — absolute sizes stay similar to landscape formats.

**Sizing ladder** (Perfect Fourth 1.333 from 20px, snapped to ×4): 20 → 28 → 36 → 48 → 64 → 84 → 112. For elements not in the table, pick the scale step that creates correct visual weight relative to the heading. All sizes should be multiples of 4.

## Engagement patterns

- Engagement typically drops after **slide 6-7**, sharper drop after slide 10
- Optimal slide count: **5-10 slides** for B2B educational content
- **First slide is the hook** — treat it like a headline. Must stop the scroll
- **Last slide should have a CTA** — follow, comment, share, visit link
- Story-driven carousels (narrative arc) earn more saves/shares than purely educational decks

## Design guidelines

### Slide structure
1. **Title slide** — bold hook, one compelling question or stat
2. **Content slides (4-8)** — one key idea per slide, large scannable text
3. **CTA slide** — clear next action

### Text
- **One key idea per slide** with large, scannable text
- **Headlines: 5-8 words** per slide
- **Body text: 2-3 short sentences** max per slide
- **Use the typography tiers table above** for sizes; min body 24px — must be readable on mobile

### Layout
- **Consistent template** across all slides — same colors, fonts, layout grid
- **Visual elements bleeding across slides** create a visual incentive to swipe
- **Z-pattern per slide**: headline top-left, visual center, page indicator bottom

### Visual consistency
- Use brand colors, fonts, and logo placement consistently across all slides
- **Template consistency drives brand recognition** — consistent branding increases recognition by 80% (University of Loyola)
- Varying content while maintaining visual identity is optimal (ScienceDirect, 1,700 messages analyzed)

### Export workflow
1. Design slides in Figma at 1080 x 1080 px (or 1080 x 1350)
2. Export as PDF — this preserves quality and is LinkedIn's preferred format
3. Upload as a document post (not image carousel)

### What works
- Educational how-to content with clear takeaways
- Data/stat breakdowns with one insight per slide
- Step-by-step processes or frameworks
- "Myths vs reality" or contrarian takes
- Company POV on industry trends with supporting data

### What to avoid
- More than 10 slides (engagement drops sharply)
- Dense text that requires reading — this is a visual medium
- Inconsistent styling across slides
- Missing CTA on the last slide
- Using it just to share a blog post — carousels should be native content
